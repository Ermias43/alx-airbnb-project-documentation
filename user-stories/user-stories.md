This document translates key use case interactions into user stories based on our use case diagram.

---

### 1. User Registration
**As a guest or host**, I want to be able to register an account so that I can book or list properties.

---

### 2. Property Search
**As a guest**, I want to search for available properties by location and date so that I can find suitable accommodations.

---

### 3. Book a Property
**As a guest**, I want to book a selected property so that I can reserve it for my trip.

---

### 4. Payment for Booking
**As a guest**, I want to securely pay for a booking so that I can confirm my reservation.

---

### 5. Manage Property Listings
**As a host**, I want to add, update, or delete my property listings so that I can control what guests see and book.

---

### 6. View Booking History
**As a user**, I want to view my booking history so that I can keep track of past stays or guest reservations.

---

### 7. Admin Management
**As an admin**, I want to manage users and listings so that I can ensure the system runs smoothly and safely.
